How to read this
🏃 Every project races its plan on the same calendar: a runner LEFT of the white TODAY line has fewer work-days done than the plan expected. 🏁 marks the last planned task of the month, │ lines are the Sunday weekly deadlines.
Every % counts worked days, not tasks: a finished task earns all of its planned days, one in progress earns the days worked so far, one not started earns 0. Example: 5 drywall days planned, 3 worked = 60%. Finish a task early and it still earns its full planned days — complete the whole month plan and the month reads 100%, even in fewer days. Task counts ("8 tasks · 3 done") are context only. Work = Action + Critical Path + City Inspections; admin tasks (meetings, checklists, walkthroughs) don't count.
The status pill judges THE CURRENT WEEK (Mon–Sun): what this week's plan demands by today — including trades that must start this week — and whether the project is on that pace. The month %, Slip % and month-end forecast are separate month-level metrics. A trade that starts this week and ends later shows as START (it passes the week once it actually starts); one finishing this week that started earlier shows as FINISH.
Pill colors: green = on pace for this week · blinking orange = behind, but the week can still be saved if the rescheduled tasks land by Sunday · yellow = slightly behind this week · blinking red = this week's plan won't hold · gray = no work scheduled this week. The "Last week" chip is the frozen Sunday-night verdict — see the LAST WEEK · REPORT view.
